# 홍보문구 skill (vibe-onestop 4단계 ②)

입력: `get_listing` JSON 중 **selling_points(검수완료분)·location_points·market·register·deal·features만 사용.
`contacts` 그룹은 입력에서 통째로 제외한다.** (연결규약 5절)

## 사전 게이트

1. `check_ad_ready` 누락 0건 확인 — 누락 있으면 문구 생성 중단하고 누락 목록 안내
2. status가 검증완료 이상인지 확인 (접수 상태 매물은 초안 표기)

## 의무기재 체크리스트 (문구에 반드시 포함)

- [ ] 소재지 (읍면동까지, 상세지번은 매체 정책 따름)
- [ ] 면적: **대장 전용면적 ㎡ 기준** (평은 괄호 병기). 복층 등 대장 외 면적은 "복층 별도(대장 미포함)" 명시
- [ ] 가격: 매매가 / 보증금·월세 (만원 단위 표기 허용, 저장은 원)
- [ ] 관리비 (정액 여부)
- [ ] 주차 가능 여부
- [ ] 입주가능일 / 임차 승계 조건
- [ ] 용도 (대장 용도 그대로 — 예: 제2종근린생활시설)
- [ ] 중개사무소 명칭·등록번호·연락처 (~/.vibe/office.json)

## 금지·주의 표현

| 금지 | 대체 |
|---|---|
| 확정 수익 보장 ("수익률 10% 보장") | "현 임대 기준 연 임대료 ○○만원 (매매가 대비 ○%)" 실계산 표기 |
| 최상급 단정 (최고·최저·유일·급매 남발) | 팩트 비교 ("반경 120m 카페 5곳") |
| 대장과 다른 면적·용도 표기 | 대장값 + 현황 병기 |
| 미검증 주장 단정 (보증보험 가능 등) | "요건 충족 예상(공시가 기준)" 수준 |
| 임차인·의뢰인 개인정보 | 절대 미출력 |

claim(접수 주장)과 verified(검증값)가 다르면 **항상 verified 기준**으로 쓴다.

## 이실장 양식

### setMemo (40자 이내, 한 줄 훅)
```
[역세권] 하단역 3분 복층 코너상가, 임차승계 연8%
```
- 구조: [키워드] + 핵심입지 + 상품형태 + 수익/가격 훅. 40자 초과 시 뒤에서부터 삭감.

### setDetailMemo (1000자 이내)
```
■ 물건 개요
 소재지·건물명·호수 / 용도(대장) / 전용면적 ㎡(평) / 층·구조

■ 거래 조건
 가격 / 관리비 / 융자 / 임차 승계 조건(보증금·월세·만기)

■ 입지 (location_points 그대로, 도보시간 포함)
 역·편의점·마트·관공서 등 3~5개

■ 추천 포인트 (selling_points 검수완료분만, 번호 목록)

■ 기타
 입주시기 / 사진·현장확인 안내

■ 중개사무소 의무기재 블록 (office.json)
```
- 단위: 가격 만원, 관리비 원 (이실장 체크리스트 v0.2 매핑표 준수)
- 전세는 `setLeasePrc` 필드 사용

## 출력 형식

의무기재 체크리스트(- [x] 완료 표시) → setMemo 안 2개 → setDetailMemo 전문 순으로 제시하고 **사람 검수를 요청**한다. 검수 전 RPA 입력 금지 (사람개입 3지점 ②).
