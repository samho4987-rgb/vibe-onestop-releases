# 블로그 skill (vibe-onestop 4단계 ③)

출력물은 `vibe_blogpost.py`가 파싱하는 원고 md (연결규약 6.2). **`## 제목(안)`과 `## 본문` 섹션 필수**, 매물글은 `## 발행 전 의무기재` 체크리스트 섹션 필수 (미완료 `- [ ]` 있으면 vibe-blogpost가 차단).

입력 제한: selling_points(검수완료)·location_points·market·register·deal·features. **contacts 제외, 본문에 전화번호·개인정보 금지** (문의는 "블로그 쪽지/사무소 방문" 유도 또는 office.json 사무소 대표번호만).

## 제목 규칙 (네이버 SEO)

- 형식: `[지역명] 핵심키워드 + 상품 + 차별점` (25~35자)
- 지역명은 검색 단위로 (예: "하단역", "사하구 하단동") — 광역시부터 쓰지 않는다
- 숫자·구체 팩트 포함 (도보 3분, 전용 26㎡)
- 제목안 3개 생성 → 사람이 택1

## 목차·본문 템플릿

```
## 제목(안)
1안 / 2안 / 3안

## 본문
[도입 2~3문장 — 지역·상권 이야기로 시작, 매물 자랑으로 시작하지 않는다]

### 이런 분께 맞습니다
[추천업종·타깃 2~3줄, features.추천업종 활용]

### 물건 개요
표: 소재지(동까지)/용도(대장)/전용면적/층·구조/가격/관리비/주차/입주

### 입지 팩트
location_points를 도보시간 포함 목록으로. 지도 캡처 이미지 자리 표시 [이미지: 지도]

### 수익 구조 (수익형 매물만)
verified 실계산만 표기. claim 수치는 "매도인 제시" 명시 후 실계산 병기

### 추천 포인트
selling_points 검수완료분. 각 포인트에 근거 한 줄

### 확인해두실 것
단점·유의사항 1~2개 솔직 표기 (신뢰 확보 — 예: 복층은 대장 미포함)

[마무리 + 문의 유도 문장. 전화번호 금지]

## 발행 전 의무기재
- [x] 소재지 / - [x] 면적(대장) / - [x] 가격 / - [x] 관리비 / - [x] 주차
- [x] 입주시기 / - [x] 용도 / - [ ] 중개사무소 (office.json 있으면 자동 충족)
```

## 문체

- 연결 폴더 `블로그_스타일가이드.md` 있으면 최우선 적용
- 기본: 존댓말, 한 문단 3문장 이내, 과장 형용사 대신 숫자
- 기사·타 블로그 문장 전재 금지

## 발행 흐름

md 저장 → `python3 vibe_blogpost.py --draft 원고.md --images 폴더/` → 임시저장 → **발행은 사람** (하루 1~2건).
